# Навигация по курсу

Этот урок поможет вам разобраться в структуре платформы, чтобы вы могли эффективно использовать все доступные инструменты обучения.

## Структура урока

Каждый урок курса следует единому шаблону:

1. **Введение** -- почему эта тема важна, какую проблему решает
2. **Основной материал** -- теория, фреймворки, определения с примерами
3. **Проверки понимания (KnowledgeCheck)** -- встроенные мини-вопросы прямо в тексте урока для самопроверки
4. **Практический раздел** -- код-примеры, инструменты, конфигурации
5. **Сценарий** -- применение концепций к одной из трёх компаний-кейсов
6. **Итоги** -- ключевые выводы и связь со следующим уроком

Не все уроки содержат каждый из этих элементов, но структура остаётся последовательной.

## Боковая навигация

В левой части экрана расположено меню модулей. Каждый модуль раскрывается в список уроков. Текущий урок выделен в навигации, и вы можете переходить к любому уроку в любом порядке.

Рекомендуемый подход -- следовать последовательности модулей, так как каждый модуль опирается на концепции предыдущих. Однако если вы уже знакомы с основами Data Governance, можете начать с интересующего вас модуля.

## Проверки понимания (KnowledgeCheck)

Внутри каждого урока вы встретите блоки **KnowledgeCheck** -- это мини-вопросы для самопроверки. Они появляются сразу после объяснения концепции и помогают убедиться, что вы поняли материал, прежде чем двигаться дальше.

Проверки понимания:
- **Не оцениваются** -- это инструмент для вас, не для системы оценок
- **Мгновенная обратная связь** -- ответ виден сразу после раскрытия
- **Не дублируют квизы** -- вопросы в квизах проверяют другие аспекты

## Квизы

В конце каждого урока (кроме ориентационных уроков M00) расположен **квиз** -- набор из 3-5 вопросов с автоматической проверкой:

- **Multiple choice** -- выберите один правильный ответ из 4 вариантов
- **True/False** -- определите, является ли утверждение истинным или ложным
- **Code challenge** -- напишите код, который проходит автоматические тесты

После ответа на каждый вопрос вы увидите подробное объяснение -- даже если ответили правильно. Объяснения содержат дополнительный контекст и углубляют понимание.

Результаты квизов **отслеживаются** и влияют на ваш общий прогресс по курсу.

## Код-челленджи

Некоторые квизы содержат **код-челленджи** -- практические задания с автоматической проверкой. Доступные языки:

| Язык | Где используется | Среда выполнения |
|------|-----------------|------------------|
| Python | Скоринг, классификация, валидация | Pyodide (в браузере) |
| SQL | Запросы к схемам, quality checks | sql.js (SQLite в браузере) |
| JSON | Валидация политик, конфигураций | JSON validator |
| YAML | Конфигурации инструментов | YAML validator |
| JavaScript | Обработка событий, трансформации | JS sandbox |

Код выполняется **полностью в браузере** -- никаких локальных установок не требуется. Вы пишете код в встроенном редакторе, нажимаете "Проверить", и тесты запускаются мгновенно.

Каждый код-челлендж содержит:
- **Стартовый код** -- шаблон с подсказками
- **Тесты** -- видимые (вы знаете, что проверяется) и скрытые (проверяют edge cases)
- **Решение** -- доступно после прохождения или после нескольких попыток
- **Подсказку** -- направляет мышление, не раскрывая ответа

## Модульные экзамены

В конце каждого модуля (кроме M00) расположен **экзамен** -- 8-12 аналитических вопросов, требующих синтеза знаний из всех уроков модуля.

Экзамены отличаются от квизов:
- **Вопросы охватывают несколько уроков** -- требуется применение знаний из разных частей модуля
- **Преобладают аналитические вопросы** -- не "что это?", а "почему?" и "как решить?"
- **Сценарии из компаний-кейсов** -- реалистичные ситуации для DataTech, BioGenesis и FinSecure
- **Порог прохождения -- 70%** -- экзамен считается сданным при набранных 70% и более

Объяснения к экзаменационным вопросам содержат ссылки на конкретные уроки модуля, где раскрыта тема.

## Прогресс и достижения

Панель прогресса показывает:
- **Процент завершения** каждого модуля
- **Результаты квизов** по каждому уроку
- **Статус экзаменов** -- сдан / не сдан
- **Общий прогресс** по курсу

Завершение всех модулей и экзаменов открывает возможность получить **сертификат** об окончании курса.

## Итоги

- Уроки следуют единому шаблону: введение, теория, практика, сценарий, итоги
- **KnowledgeCheck** внутри уроков -- для самопроверки, не оцениваются
- **Квизы** в конце уроков -- оцениваются, 3-5 вопросов включая код-челленджи
- **Экзамены** в конце модулей -- 8-12 аналитических вопросов, порог 70%
- Код-челленджи выполняются **в браузере** на Python, SQL, JSON, YAML, JavaScript
- Панель прогресса отслеживает завершение уроков, квизов и экзаменов
